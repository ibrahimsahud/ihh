- Farkli bir sunucu adi veya instance (ornegin IBOO\SQLEXPRESS)
  kullaniyorsaniz database.js ve server.js dosyalarini guncelleyin.
- TCP/IP protokolunun SQL Server Configuration Manager uzerinden
  etkin oldugundan emin olun.

  alanlarini doldurun ve sunucu adindaki \\ kismini gerekirse duzenleyin.

-----------------------------------------------------------
Sorun giderme
-----------------------------------------------------------

Hata: "ConnectionError: Failed to connect to IBOO"
- SQL Server servis durumu Running mi?
- SSMS uzerinden dogru sunucu adini kullanin.
- TCP/IP etkinken tekrar deneyin.
- Servisi yeniden baslatin.

Hata: "Login failed for user 'IBOO\\iboo'"
- Kullanici yetkilerini kontrol edin.
- Veritabanina kullaniciyi ekleyip db_owner yetkisi verin.

Hata: "Cannot open database 'IHH_Hayir'"
- `database-schema.sql` dosyasini calistirarak veritabani olusturun.
- SSMS uzerinde adin dogru yazildigini dogrulayin.
